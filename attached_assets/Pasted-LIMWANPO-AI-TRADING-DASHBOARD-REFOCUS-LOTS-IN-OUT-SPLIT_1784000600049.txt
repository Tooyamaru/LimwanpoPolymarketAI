STEP 3 — ROLLOVER ISOLATION
==================================================

Ketika market rollover:

old condition_id:
- tidak lagi tampil sebagai current card
- tetap ada di history
- tetap diproses oleh exit/resolution/outcome learning
- jika masih OPEN setelah expiry:
  status semantic = EXPIRED / RESOLUTION_PENDING
- tidak dihitung ke current card LOTS/PNL/IN/OUT

new condition_id:
- menjadi card aktif
- summary dimulai dari posisi condition_id baru saja
- tidak mewarisi posisi lama

Buat pemisahan query:

CURRENT CARD SUMMARY:
- active condition_id only

OPEN LEGACY POSITIONS:
- old condition_id masih OPEN/PARTIAL

TRADE HISTORY:
- all condition_ids

PORTFOLIO TOTAL:
- all relevant positions

Jangan memindahkan data lama secara palsu.
Jangan mengganti condition_id position lama.

STEP 4 — LOTS ON EACH CARD
==================================================

Tambahkan LOTS pada setiap card.

LOTS berarti:

jumlah OPEN + PARTIAL position lots
untuk current condition_id

Fields:

- open_lot_count
- partial_lot_count
- active_lot_count

Formula:

active_lot_count =
open_lot_count + partial_lot_count

Display:

LOTS 0

atau:

LOTS 3 OPEN

Jika mixed:

LOTS 3
2 OPEN · 1 PARTIAL

Jangan menghitung CLOSED lots ke LOTS aktif.

STEP 5 — SPLIT IN / OUT DISPLAY
==================================================

Pisahkan count dan amount secara jelas.

IN:
- entry_count
- entry_amount_usdc

OUT:
- exit_count
- exit_amount_usdc

IN
4x
$40.00

OUT
2x

Jika layout terlalu sempit:

IN 4x · $40.00

Aturan:

IN count:
- executed OPEN_LONG_YES
- executed OPEN_LONG_NO
- scale-in executed
- re-entry executed

IN amount:
- sum actual entry_price × quantity

OUT count:
- executed partial exit
- executed final exit
- executed expiry exit

OUT amount:
- sum actual exit_price × close_quantity

Jangan menghitung:

- WAIT
- BLOCKED
- REJECTED
- FAILED
- PENDING
- duplicate retry

==================================================
STEP 6 — LIVE PNL RULE
==================================================

PNL card harus nyata.

Formula:

total_pnl =
realized_pnl + unrealized_pnl

Per current condition_id only.

realized_pnl:
- sum dari CLOSED/PARTIAL exit events current condition_id

unrealized_pnl:
- sum open remaining lots current condition_id
- gunakan executable exit price real

LONG_YES:
- current YES bid

LONG_NO:
- current NO bid
- jika derived:
  no_bid = 1 - yes_ask

Expired/resolved:

- gunakan forced_exit_price / direct resolution
- jangan pakai stale CLOB

Jika no position:

PNL —

Jika current price missing:

PNL N/A

Jangan klaim $0.00 jika unrealized tidak bisa dihitung.
