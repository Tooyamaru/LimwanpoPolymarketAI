STEP 4 — SIGNAL ENGINE TRACE
==================================================

raw CLOB snapshot
→ derived features
→ signal type
→ severity
→ direction
→ confidence formula
→ final confidence_score
→ DB signal row

- terlalu konstan
- memakai fallback
- memakai stale snapshot
- memakai default severity
- mengabaikan market-specific input

Jika sama karena input memang identik, dokumentasikan.
Jika sama karena bug, perbaiki.

==================================================
STEP 9 — NEWS ENGINE
==================================================

News Engine saat ini diduga stub NEUTRAL / 0.0.

Audit dan jawab:

- Apakah output stub masuk ke Decision Engine?
- Apakah Decision Engine memperlakukan stub sebagai data nyata?
- Apakah stub menurunkan confidence?
- Apakah stub memblokir keputusan?
- Apakah stub hanya bersifat netral dan tidak berpengaruh?

Jika stub memengaruhi keputusan:
itu INVALID.

Perbaiki agar stub tidak dianggap sebagai evidence.

Jangan tambah provider berita baru.
Gunakan nullable/not_available state jika data belum ada.

==================================================
STEP 10 — DYNAMIC WEIGHT
==================================================

Audit default weight 50.0.

Jawab:

- Kapan dipakai?
- Mengapa 50.0?
- Apakah memengaruhi keputusan sebelum learning tersedia?
- Apakah dianggap learned weight padahal belum?
- Apakah UI/API membedakan default vs learned?

Jika default dipakai sebagai learned evidence:
perbaiki.

Harus ada status jelas:

- DEFAULT
- LEARNED
- INSUFFICIENT_HISTORY
