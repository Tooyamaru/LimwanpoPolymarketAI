The system trades:

* YES contracts
* NO contracts

on:

Polymarket prediction markets.

Examples:

* Will BTC be above X at Y time?
* Will ETH be above X at Y time?
* Will SOL be above X at Y time?
* Will XRP be above X at Y time?

# ROLE OF BINANCE AND CHAINLINK

They are used only for:

* Signal generation
* Market analysis
* Probability estimation
* Quantitative modeling

The system never executes trades on Binance.

The system never opens futures positions.

The system never opens spot positions.

The system never sends orders to a centralized exchange.

All execution occurs exclusively on Polymarket.

# QUANT PHILOSOPHY

LimwanpoPolymarketAI is NOT a direction-prediction bot.

The goal is NOT:

"Predict if BTC goes up."

The goal IS:

"Detect probability mispricing."

Example:

Market YES probability:
95%

Model probability:
80%

Even if the model believes the event is likely,

BUY YES may still be a bad trade.

The system must evaluate:

Model Probability

versus

Market Implied Probability

and determine whether YES or NO is mispriced.

This principle should be documented clearly.

---

# POSITION PHILOSOPHY

Multiple entries are allowed.

Multiple positions on the same market may be valid.

Multiple positions on the same condition_id may be valid if exposure rules permit.

Do NOT assume:

one market = one position.

Do NOT automatically classify repeated entries as bugs.

Exposure management is more important than position count.

Any documentation implying a strict one-position-per-market rule should be reviewed and corrected.

---

# PROFIT TAKING PHILOSOPHY

The system is allowed to close positions before market settlement.

Waiting for event resolution is NOT mandatory.

Valid workflow:

OPEN YES
→ position becomes profitable
→ CLOSE POSITION
→ realize profit

This is expected behavior.

This is not an error.

This is not a failed prediction.

This is a normal Polymarket trading workflow.

Documentation must clearly explain this distinction.
