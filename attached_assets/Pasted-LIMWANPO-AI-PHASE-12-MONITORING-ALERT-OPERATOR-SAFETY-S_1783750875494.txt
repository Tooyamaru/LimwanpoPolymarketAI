STEP 3 — DEFINE CRITICAL ALERTS
===============================

Buat daftar alert wajib berbasis real DB / real engine state.

Minimal alert:

1. ENGINE_STALLED

   * jika worker tertentu tidak update lebih dari X menit

2. GAMMA_API_DEGRADED

   * jika Gamma calls fail berulang

3. CLOB_API_DEGRADED

   * jika CLOB/orderbook fetch fail berulang

4. NO_MARKETS_ACTIVE

   * jika market_universe active count = 0

5. EXPIRED_POSITION_OPEN

   * jika position.status=OPEN dan market.end_time < now

6. DIRECT_RESOLUTION_MISSING

   * jika market expired tapi outcome_learning outcome_source masih NOT_AVAILABLE terlalu lama

7. EXECUTION_ERROR_SPIKE

   * jika trade_decisions FAILED meningkat

8. RISK_GATE_STUCK

   * jika risk_gated=true terlalu lama / terlalu banyak secara tidak normal

9. DUPLICATE_POSITION

   * jika lebih dari 1 OPEN position untuk condition_id sama

10. PORTFOLIO_EXPOSURE_HIGH

* jika exposure > configured cap

11. PAPER_PNL_ANOMALY

* jika realized/unrealized PnL impossible / null pada closed position

12. FORCED_EXIT_PENDING

* jika expired open position belum punya CLOSE_POSITION decision

Untuk setiap alert, dokumentasikan:

* query / condition
* severity
* threshold
* recommended operator action
* false positive risk

Severity:

* INFO
* WARNING
* CRITICAL
