==================================================
STEP 3 — DEFINE CANONICAL MARKET STATE
==================================================

Buat satu fungsi canonical, misalnya:

get_market_lifecycle_state(market, now)

Output wajib:

- PRE_MARKET
- ACTIVE
- EXPIRED
- RESOLUTION_PENDING
- RESOLVED
- INVALID_TIME_STATE

Aturan dasar:

PRE_MARKET:
- now < start_time
- market belum boleh menghasilkan entry decision
- execution_allowed = false

ACTIVE:
- start_time <= now < end_time
- market active
- closed=false
- execution_allowed = true jika semua gate lain lolos

EXPIRED:
- now >= end_time
- belum ada direct resolution
- execution_allowed = false

RESOLUTION_PENDING:
- market expired/closed
- direct resolution belum tersedia
- execution_allowed = false

RESOLVED:
- direct Polymarket resolution tersedia
- execution_allowed = false
- outcome learning boleh berjalan

INVALID_TIME_STATE:
- timestamp hilang/kontradiktif
- execution_allowed = false

Gunakan UTC secara konsisten.

STEP 5 — BACKEND AI GATING
==================================================

Audit Signal, Opportunity, Strategy, Risk, dan Decision Engine.

Aturan:

PRE_MARKET:
- boleh simpan seed/orderbook/reference snapshot sebagai observasi
- jangan menghasilkan BUY_YES / BUY_NO / OPEN_LONG
- decision harus NOT_ACTIVE / PRE_MARKET
- confidence keputusan aktif harus null / NOT_AVAILABLE
- opportunity boleh diberi classification PRE_MARKET_PREVIEW jika dibutuhkan
- jangan dicampur dengan active opportunities

ACTIVE:
- engine berjalan normal
- signal/confidence/opportunity/strategy/risk/decision dihitung dari live data

EXPIRED / RESOLUTION_PENDING / RESOLVED:
- jangan membuat entry decision baru
- hanya exit/resolution/outcome-learning path yang boleh berjalan

Cari dan perbaiki semua query yang mengambil market tanpa lifecycle filter.

==================================================
STEP 6 — EXECUTION SAFETY GATE
==================================================

Execution Engine wajib revalidate sebelum membuat position:

- lifecycle_state == ACTIVE
- start_time <= now
- end_time > now
- market closed=false
- market active=true jika field reliable
- condition_id/token IDs valid
- decision timestamp masih fresh

Jika tidak:
- reject dengan reason:
  - MARKET_NOT_STARTED
  - MARKET_EXPIRED
  - MARKET_CLOSED
  - INVALID_MARKET_TIME
  - STALE_DECISION

Jangan hanya percaya status dari Decision Engine.
Execution harus melakukan revalidation sendiri.
