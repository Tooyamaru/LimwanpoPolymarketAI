LIMWANPO AI — UNLIMITED POSITION COUNT + FAST PROFIT EXIT

Konteks:

Requirement baru:

1. Jumlah posisi/lot tidak boleh dibatasi oleh fixed position count.
2. AI boleh membuka berapa pun lot selama modal dan exposure masih aman.
3. Posisi tidak perlu menunggu market selesai.
4. Jika setelah 2–5 detik atau kapan pun posisi sudah menghasilkan profit bersih, AI boleh keluar.
5. Tetap paper trading.
6. Jangan redesign dashboard.

PENTING:

UNLIMITED berarti:

- tidak ada MAX_OPEN_POSITIONS count cap
- tidak ada PORTFOLIO_MAX_OPEN_POSITIONS count cap
- tidak ada MAX_ENTRIES_PER_MARKET count cap
- tidak ada MAX_OPEN_LOTS_PER_MARKET count cap

Tetapi tetap wajib ada:

- available capital check
- total exposure check
- per-market exposure check
- per-asset exposure check
- entry size bounds
- duplicate execution protection
- cooldown atau signal-change protection
- no repeated identical entry spam

JANGAN membuat jumlah posisi unlimited sekaligus menghapus seluruh risk management.

==================================================
PART 1 — REMOVE FIXED POSITION COUNT LIMITS
==================================================

Audit seluruh code:

- PORTFOLIO_MAX_OPEN_POSITIONS
- MAX_OPEN_POSITIONS
- MAX_ENTRIES_PER_MARKET
- MAX_OPEN_LOTS_PER_MARKET
- MAX_SAME_SIDE_ENTRIES
- PORTFOLIO_POSITION_LIMIT
- MAX_ENTRIES
- MAX_LOTS

Fixed count limits tersebut tidak boleh lagi memblokir entry.

Replace semantic:

OLD:
if open_positions >= 5:
    BLOCK

NEW:
if available_capital < required_notional:
    BLOCK

if total_exposure + proposed_notional > exposure_limit:
    BLOCK

if market_exposure + proposed_notional > market_exposure_limit:
    BLOCK

if asset_exposure + proposed_notional > asset_exposure_limit:
    BLOCK

Tidak boleh ada fixed limit seperti 5, 10, atau 20 position rows.

==================================================

Pertahankan atau tambahkan settings:

- MAX_TOTAL_EXPOSURE_USDC
- MAX_EXPOSURE_PER_MARKET_USDC
- MAX_EXPOSURE_PER_ASSET_USDC
- MAX_ENTRY_NOTIONAL_USDC

Contoh:

available_capital =
initial_capital
+ realized_pnl
- current_open_exposure

Entry allowed jika:

available_capital - proposed_notional
>= minimum reserve

Jangan menggunakan position count sebagai proxy exposure.

==================================================
PART 3 — PREVENT ENTRY SPAM
==================================================

Walaupun position count unlimited, jangan membuat lot baru setiap worker cycle.

Entry tambahan harus mempunyai alasan baru:

- opportunity score improved
- price improved
- confidence improved
- order flow confirmation
- signal changed
- partial allocation continuation
- re-entry after exit

Identical signal + identical price + identical score:

BLOCK:
SCALE_IN_NO_IMPROVEMENT

Duplicate decision/order:

BLOCK:
DUPLICATE_EXECUTION

Jangan hardcode 2 atau 5 detik di engine.
Gunakan settings.

==================================================
PART 5 — EXECUTABLE EXIT PRICE
==================================================

Profit exit wajib memakai harga yang benar-benar bisa dieksekusi.

LONG_YES / UP:

exit_price = current YES bid

LONG_NO / DOWN:

exit_price = current NO bid

Jika NO bid diturunkan:

no_bid = 1 - yes_ask

Jangan memakai:

- midpoint
- yes_mid
- target price
- ask untuk menjual
- stale price

Jika orderbook stale/missing:

- jangan exit normal
- return PRICE_NOT_AVAILABLE / STALE_ORDERBOOK

==================================================
PART 6 — NET PROFIT FORMULA
==================================================

Gross PnL:

gross_pnl =
(exit_price - entry_price)
× close_quantity

Estimated costs:

- spread impact
- slippage estimate
- configured paper fee jika ada

Net PnL:

net_pnl =
gross_pnl
- estimated_costs

Fast exit hanya jika:

net_pnl >= MIN_NET_PROFIT_AFTER_COST_USDC

dan salah satu:

gross_pnl >= FAST_PROFIT_TARGET_USDC

atau

profit_percent >= FAST_PROFIT_TARGET_PERCENT

Jangan exit karena gross profit kecil tetapi setelah spread sebenarnya rugi.

==================================================
PART 7 — HOLD TIME
==================================================

Position boleh keluar setelah:

now - opened_at >= MIN_POSITION_HOLD_SECONDS

Jika target profit tercapai dalam 2 detik dan minimum hold = 2:

boleh keluar.

Jika target tercapai dalam 5 detik:

boleh keluar.

Tidak perlu menunggu market expiry.

Market expiry hanya menjadi fallback force-close.

==================================================
PART 8 — PARTIAL VS FULL FAST EXIT
==================================================

Buat aturan configurable:

FAST_EXIT_MODE:

- FULL
- PARTIAL
- DYNAMIC

FULL:
- close seluruh remaining quantity

PARTIAL:
- close configured percentage

DYNAMIC:
- profit kecil → partial
- profit kuat → full

Contoh:

FAST_EXIT_PARTIAL_PERCENT = 50

Pastikan:

- partial exit mengurangi remaining_quantity
- OUT count bertambah
- realized PnL benar
- lot masih OPEN/PARTIAL jika tersisa
- final exit menutup lot

Jangan membuat profit exit mengalahkan emergency/risk exit.

PART 11 — CONCURRENT ENTRY SAFETY
==================================================

Target:

Tidak perlu membatasi jumlah rows, tetapi total notional harus aman.
