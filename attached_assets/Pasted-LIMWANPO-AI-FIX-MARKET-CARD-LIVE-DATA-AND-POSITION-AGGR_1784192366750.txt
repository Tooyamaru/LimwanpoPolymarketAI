2. FIX UP AND DOWN LIVE PRICES
==================================================

UP and DOWN prices must come from the latest fresh Polymarket CLOB orderbook for the exact active condition_id.

Display:

UP mark =
(best_up_bid + best_up_ask) / 2

DOWN mark =
(best_down_bid + best_down_ask) / 2

If the project only receives the UP orderbook:

down_bid = 1 - up_ask
down_ask = 1 - up_bid
down_mark = (down_bid + down_ask) / 2

For PnL valuation:

LONG_UP uses executable UP bid.
LONG_DOWN uses executable DOWN bid.

Do not use:
- fixed 0.505 / 0.495
- seed price after a fresh orderbook exists
- stale opportunity row
- position entry price as current market price
- default 0.5 silently

Add freshness validation.

If the latest snapshot exceeds the configured freshness limit:

- show STALE
- keep last trusted value if appropriate
- do not pretend the price is live

==================================================
3. FIX SIDE HIGHLIGHT
==================================================

For each active card, calculate:

up_open_exposure =
SUM(remaining_quantity × lot_entry_price)
WHERE condition_id = active condition
AND side = LONG_UP / LONG_YES
AND status IN ('OPEN', 'PARTIAL')

down_open_exposure =
SUM(remaining_quantity × lot_entry_price)
WHERE condition_id = active condition
AND side = LONG_DOWN / LONG_NO
AND status IN ('OPEN', 'PARTIAL')

Visual rules:

- no UP and no DOWN position:
  both dim

- UP exposure > 0 and DOWN exposure = 0:
  UP bright
  DOWN dim

- DOWN exposure > 0 and UP exposure = 0:
  DOWN bright
  UP dim

- both exposures > 0:
  both active
  larger exposure receives stronger emphasis

When remaining_quantity becomes 0:
- that side becomes inactive immediately

Highlight must not depend on the latest AI prediction.

==================================================
4. DEFINE ENTRY AND EXIT METRICS
==================================================

For each active condition and side:

entry_fill_count =
COUNT FILLED BUY orders

entry_notional =
SUM(entry_fill_quantity × entry_fill_price)

exit_fill_count =
COUNT FILLED SELL orders

exit_proceeds =
SUM(exit_fill_quantity × exit_fill_price)

open_lot_count =
COUNT lots WHERE remaining_quantity > 0
AND status IN ('OPEN', 'PARTIAL')

open_shares =
SUM(remaining_quantity)

open_cost_basis =
SUM(remaining_quantity × lot_entry_price)

weighted_average_entry =
open_cost_basis / open_shares

realized_pnl =
SUM realized PnL from closed quantities

unrealized_pnl =
SUM(
  remaining_quantity ×
  (executable_side_bid - lot_entry_price)
)

Card labels:

LOTS = open_lot_count
IN = entry_fill_count + total entry_notional
OUT = exit_fill_count + total exit proceeds

Do not use entry_count as open_lot_count.
Do not use total original quantity after partial exits.
Do not count rejected, pending, cancelled, or unfilled orders.
