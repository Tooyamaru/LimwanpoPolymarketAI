6. DEFINE IN, OUT, AND LOTS
==================================================

Use these exact definitions.

ENTRY FILLS / IN COUNT:

entry_fill_count =
COUNT FILLED BUY orders
for the exact condition_id

ENTRY NOTIONAL / IN AMOUNT:

entry_notional =
SUM(filled buy quantity × filled buy price)

EXIT FILLS / OUT COUNT:

exit_fill_count =
COUNT FILLED SELL orders
for the exact condition_id

EXIT PROCEEDS / OUT AMOUNT:

exit_proceeds =
SUM(filled sell quantity × filled sell price)

OPEN LOTS:

open_lot_count =
COUNT position lots
WHERE remaining_quantity > 0
AND status IN ('OPEN', 'PARTIAL')
AND condition_id = exact active condition_id

OPEN SHARES:

open_shares =
SUM(remaining_quantity)

OPEN COST BASIS:

open_cost_basis =
SUM(remaining_quantity × entry_price)

AVERAGE ENTRY:

average_entry =
open_cost_basis / open_shares

Card labels:

LOTS:
open_lot_count

IN:
entry_fill_count × and total entry_notional

OUT:
exit_fill_count × and total exit proceeds

Do not:

- count PENDING orders
- count REJECTED orders
- count CANCELLED orders
- use original quantity after partial exit
- treat entry count as open lots
- treat position rows and order fills as the same metric

10. COUNTDOWN COMPUTATION
==================================================

Backend source of truth:

countdown_seconds =
max(
    0,
    floor(market_end_time_utc - server_time_utc)
)

Frontend should not independently invent the expiry.

Frontend uses:

- server_time returned by API
- market_end_time returned by API
- local monotonic countdown between polling cycles
- re-synchronization on every API poll

Recommended frontend model:

server_offset_ms =
api_server_time_ms - browser_time_at_response_ms

display_now =
Date.now() + server_offset_ms

remaining =
market_end_time_ms - display_now

Prevent local computer clock drift from corrupting countdown.

Always use UTC internally.

Only format to MM:SS for display.
